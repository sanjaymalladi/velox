# Rendering

Rendering walks the timeline frame by frame and writes the resulting animation to disk in your chosen format.

## Output Formats

Velox supports multiple output formats to suit different use cases:

| Format | Best For | File Extension |
|--------|----------|----------------|
| `mp4` | High quality video, web, social media | `.mp4` |
| `webm` | Smaller file size, web applications | `.webm` |
| `gif` | Simple animations, compatibility | `.gif` |

## Quality Settings

### CRF (Constant Rate Factor)

Use the `--quality` flag to control output quality:

- **`low`** (CRF 23-25): Smaller file size, lower quality
- **`medium`** (CRF 20-22): Balanced quality and file size
- **`high`** (CRF 18-19): High quality, larger file size

```bash
# High quality MP4 render
velox-video render animation.ts --quality high --output animation.mp4

# Medium quality WebM render
velox-video render animation.ts --quality medium --format webm --output animation.webm
```

## Advanced Rendering Options

### Frame Rate Control

Override the project's default FPS:

```bash
velox-video render animation.ts --fps 60
```

### Resolution

Set custom output dimensions:

```bash
velox-video render animation.ts --width 1280 --height 720
```

### Looping

Repeat the animation multiple times:

```bash
# Loop 3 times
velox-video render animation.ts --loop --count 3
```

### Preset Options

Choose encoding preset for speed vs compression tradeoff:

```bash
# Fast encoding (larger file)
velox-video render animation.ts --preset fast

# Slow encoding (smaller file)
velox-video render animation.ts --preset slow
```

## Performance Tips

1. **Use `--quality medium`** for faster renders with good quality
2. **Lower resolution** when testing to save time
3. **Use `--preset fast`** during development, `--preset slow` for final output
4. **Enable parallel processing** if supported by your system
5. **Close unnecessary applications** to free up memory for rendering
6. **Test with low quality first**, then re-render with high quality

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Out of memory | Reduce resolution or quality setting |
| Slow render | Check if WebAssembly is properly compiled, use `--preset fast` |
| Corrupted output | Verify source animation doesn't have errors |
| Wasm compilation error | Ensure WASM support is enabled in browser/node |
| Black output | Check animation timing and visibility |
| Format not working | Ensure ffmpeg/lib dependencies are installed |

2. Test different parameters one at a time
3. Compare outputs side by side
4. Document settings that work well

```bash

velox-video render anim.ts --quality low --output quick-test.mp4
# Review, then improve
velox-video render anim.ts --quality high --output final.mp4
```
