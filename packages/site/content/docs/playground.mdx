# Playground

The playground is the fastest place to try the current Velox API without leaving the browser. It is best used for quick experiments, layout checks, and validating small scenes before you move back into a real `video.ts` file.

## What It Is Good For

- Testing builder syntax quickly
- Comparing animation presets
- Exploring aspect-ratio-safe layouts
- Verifying cards, charts, and simple scene ideas

## What It Is Not For

- Final render output
- Large multi-scene videos
- File-based project structure
- Asset-heavy production work

## Recommended Workflow

1. Start from one of the built-in examples.
2. Change one thing at a time.
3. Validate motion, spacing, and readability.
4. Move the working code into `video.ts`.
5. Use `velox preview` and `velox render` for the real workflow.

## Example: Compact Builder Scene

```ts
import { createVideo, scene, text, shape } from '@velox-video/core'

export default createVideo({
  size: '720p',
  fps: 30,
  background: '#0a0a0f',
  scenes: [
    scene(4)
      .background(shape.gradient('135deg', '#0a0a0f', '#1a0a2e'))
      .add(
        text('VELOX')
          .center({ offsetY: -30 })
          .size(120)
          .weight(900)
          .color('#ffffff')
          .in('slideUp', 0.8),
        text('Motion Graphics Engine')
          .center({ offsetY: 60 })
          .size(32)
          .color('#a78bfa')
          .in('fadeIn', 0.6, { delay: 0.4 }),
      ),
  ],
})
```

## Example: LLM-First Scene Structure

```ts
import { createExplainerVideo } from '@velox-video/core'

export default createExplainerVideo({
  title: 'How AI Agents Work',
  duration: 60,
  aspectRatio: '9:16',
  theme: 'tech',
  sections: [
    { type: 'hook', heading: 'From Prompt to Workflow' },
    { type: 'problem', heading: 'Teams Lose Time', points: ['Manual work', 'Context switching'] },
    { type: 'process', heading: 'The Loop', steps: ['Input', 'Plan', 'Execute', 'Review'] },
    { type: 'stats', heading: 'Impact', stats: [{ label: 'Time Saved', value: '68%' }] },
    { type: 'cta', heading: 'Start With Structure' },
  ],
})
```

## Tips

- Prefer explicit `size` or `aspectRatio`.
- Keep copy short while iterating.
- Check contrast before adding more effects.
- Use one visual idea per scene.
- If the result feels busy, remove elements before adding more animation.
