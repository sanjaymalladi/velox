# Preview Studio

The Preview Studio provides an interactive development environment with Hot Module Replacement (HMR) for rapid prototyping and iteration.

## Features

### Hot Module Replacement (HMR)

Any changes you make to your animation code are automatically reflected in the preview without requiring a full reload. This enables rapid development and experimentation.

### Real-time Debugging

- Live console output
- Error overlays with detailed messages
- Stack traces for runtime errors
- Performance metrics display
- FPS counter and memory usage

### Interactive Controls

- Play/Pause/Stop controls
- Speed adjustment (0.1x to 4x)
- Frame-by-frame navigation
- Timeline scrubbing
- Export buttons for quick rendering

## Interactive Playground

Try these interactive examples in the studio:

**Example 1: Interactive Color Animation**
```typescript
import { canvas, rect, color, animate } from '@velox-video/core'

export default canvas({
  width: 1920,
  height: 1080,
  fps: 30,
  duration: 5.0,
  children: [
    rect({
      width: 150,
      height: 150,
      fill: color('magenta'),
      x: animate({ from: 0, to: 300, duration: 2.0 }),
      y: animate({ from: 0, to: 300, duration: 2.0 }),
    }),
  ],
})
```

**Test Variations in Studio:**
- Change color to 'cyan' or 'yellow'
- Modify duration to 1.0 or 10.0
- Add more rectangles with different timing
- Experiment with easing functions

**Example 2: Interactive Timing Test**
```typescript
import { canvas, circle, color, animate } from '@velox-video/core'

export default canvas({
  width: 1920,
  height: 1080,
  children: [
    circle({
      radius: 50,
      fill: color('lime'),
      x: animate({ from: 100, to: 800, duration: 4.0, easing: 'easeInOut' }),
      y: animate({ from: 100, to: 800, duration: 4.0, easing: 'easeInOut' }),
    }),
  ],
})
```

**Test in Studio:**
- Use playback controls to slow down and examine
- Adjust speed to 0.5x for detailed inspection
- Use frame-by-frame navigation for precision

## Configuration

Create a preview configuration in `velox.config.ts`:

```typescript
import { defineConfig } from '@velox-video/core'

export default defineConfig({
  preview: {
    port: 3000,
    host: 'localhost',
    open: true,  // Auto-open browser
    hmr: true,   // Enable Hot Module Replacement
    quality: 'high'  // Preview quality
  }
})
```

## Development Workflow

### Step-by-Step Process

1. **Start the Studio**
   ```bash
   npx velox-video preview src/index.ts
   ```

2. **Make Code Changes**
   - Edit your TypeScript file in your editor
   - Save the file (Ctrl+S)

3. **Observe Changes**
   - HMR automatically updates the preview
   - No manual refresh needed
   - State is preserved when possible

4. **Test Variations**
   - Try different parameters
   - Compare multiple versions
   - Use playback controls to analyze

5. **Debug Issues**
   - Check console for errors
   - Use error overlays
   - Adjust parameters in real-time

6. **Export Final Version**
   - Stop the studio (Ctrl+C)
   - Run render command
   - Verify exported output

## Interactive Debugging Techniques

### Common Patterns

**Pattern 1: Iterative Refinement**
```bash
# Start with basic version
npx velox-video preview src/animation.ts

# In another terminal, test variations
npx velox-video render src/animation.ts --quality low --output test1.mp4
npx velox-video render src/animation.ts --quality high --output test2.mp4
```

**Pattern 2: Parameter Tuning**
- Adjust FPS in studio
- Test different resolutions
- Compare performance

**Pattern 3: State Preservation**
- Use HMR to maintain state
- Test edge cases
- Roll back changes easily

## Performance Optimization

### In Studio Mode
- Use lower preview quality for faster updates
- Disable unnecessary features during development
- Monitor performance metrics

### Testing Best Practices
- Test on target devices
- Verify export quality matches preview
- Document successful parameter combinations

## Troubleshooting

| Issue | Solution |
|-------|----------|
| HMR not working | Check console for errors, ensure file is saved |
| Studio slow | Reduce preview quality, disable unnecessary effects |
| Console not showing | Open browser dev tools, check network tab |
| Port conflict | Change port in config or use `--port` flag |
| Module errors | Run `npm install`, check TypeScript errors |
